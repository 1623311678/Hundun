# 什么是快应用？

快应用（Quick App）是一种基于 Web 技术（JavaScript、CSS、XML/JSON）开发的轻量级应用形态，由华为等主流手机厂商联合推出。快应用无需安装，扫码或点击即可使用，具备接近原生 App 的体验和系统能力，体积小、启动快、分发便捷，适合轻量、低频、活动类等场景。

## 快应用的主要特点

- **免安装即用**：用户无需下载安装，扫码/点击即可运行。
- **体积小巧**：一般几十 KB 到几百 KB，启动极快。
- **系统能力强**：可调用部分原生 API（如定位、相机、支付等），体验接近原生。
- **统一标准**：由快应用联盟（华为、小米、OPPO、vivo 等）共同推动，生态开放。
- **开发高效**：采用 Web 技术栈，开发门槛低，支持主流前端框架（如 Taro、uni-app）。

# 快应用 vs React Native 原生 App

| 对比项         | 快应用（Quick App）                | RN 原生 App（React Native）         |
|----------------|-----------------------------------|-------------------------------------|
| **开发语言**   | JS/CSS/XML（Web 技术）             | JS + RN 组件 + 原生桥接             |
| **运行方式**   | 容器/引擎中运行（非独立进程）      | 独立进程，JS 代码通过 Bridge 调用原生 |
| **系统能力**   | 部分原生能力，有限制               | 可桥接全部原生能力                  |
| **安装方式**   | 免安装，扫码/点击即用              | 需下载安装                          |
| **体积/启动**  | 极小/极快                          | 较大/较快                           |
| **UI体验**     | 接近原生，部分组件有限              | 真正原生组件，体验最佳               |
| **分发渠道**   | 快应用中心、扫码、分享              | 应用市场、企业分发                   |
| **适用场景**   | 轻量、低频、活动页、工具类          | 复杂、重度、深度集成场景             |

**简要总结**：  
- 快应用适合轻量、低频、拉新、活动等场景，开发快、分发便捷，但能力有限。  
- RN 原生 App 适合复杂、重度、深度集成场景，体验和能力最强，但需安装和维护。

# 业界主流快应用方案

1. **快应用联盟标准（Quick App Alliance）**  
   - 由华为、小米、OPPO、vivo 等联合推出，统一标准，生态开放。
   - 支持主流安卓手机，开发一次多端运行。

2. **鸿蒙快应用（HarmonyOS Quick App）**  
   - 华为鸿蒙系统专属快应用，兼容联盟标准，支持更多鸿蒙原生能力。

3. **Taro/uni-app 等多端框架**  
   - 支持快应用平台编译，开发者可用一套代码适配快应用、微信小程序、H5、App 等多端。

4. **小程序（如微信/支付宝/百度/抖音小程序）**  
   - 虽与快应用不同，但同属"轻应用"范畴，理念类似，生态各自独立。
