# react-antd-mobile 标准化模版

react-antd-mobile 标准化模版,整体采用 webpack+react+antd-mobile 组件库的技术方案。

## 快速开始

```shell
hundun create my-app
#如果没有安装hundun-cli可以运行sudo npm install hundun-cli -g来安装，之后再执行创建命令即可
```

## 目录结构

```javascript
｜--root//根目录
    |--src//前端资源目录
        |--api//接口
        |--components//组件
        |--pages//
              |--pageA//应用A
                    |--app.less
                    |--app.tsx//入口
                    |--router//应用A路由
              |--pageB//应用B
        |--store//状态管理，redux
        |--utils//工具包
        |--theme.less//主题
    |--config//编译配置
        |--webpack.base.config.js//基础配置
        |--webpack.dev.config.js//开发配置
        |--webpack.prod.config.js//生产环境配置
    |--public//静态资源，webpack不会打包里边的内容，需要借助插件copy到打包之后的目录。比如dist
        |--flexible.js//阿里淘宝flexible
        |--index.html
    |--babel.config.js//babel配置
    |--commitlintrc.js//commitlint
    |--.eslintrc.js//eslint
    |--.prettierrc.js//prettier
    |--tsconfig.json
    |--package.json
    |--package-lock.json
    |--README.md
```

## 内容

<img style={{ width: 500, height: 500, objectFit: "contain" }} src={antdMobilePng}/>

1、引入了 flexible.js+rem 以及postcss-px-to-viewport的移动端适配方案，可以直接写px，自动转换vw，vh

2、引入了 vconsole 来作为移动端线上调试

3、配置了 webpack-dev-server 的 host 为 0.0.0.0,在手机与电脑同一个局域网下，手机可以直接通过 IP 来访问本地页面。

4、antd-mobild-UI, 引入了 swipper，并简单做 demo

5、多页面应用架构，本地可直接打包看静态页面

6、结合当下流行的commit Lint，prettier,eslint提供代码增量检查的能力

7、webpack多页面构建，axios封装

8、修改flexable，删去对字体的处理，设置1rem🟰100px，直接写px即可，本身就是独立像素，不要写rem

9、启动步骤 ：

1）npm install

2）npm start

之后会自动打开http://0.0.0.0:9002/pageA.html#/  可以自己切换 http://0.0.0.0:9002/pageB.html#/
